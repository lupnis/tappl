# Code of Conduct

## Our Commitment

We want TAP to be a welcoming, respectful, and sustainable project for everyone.
Contributors, maintainers, and community members are expected to help create an environment that is constructive, collaborative, and harassment-free.

## Expected Behavior

- be respectful in language and tone
- assume good intent, but listen when impact is pointed out
- give feedback about code and ideas, not about people
- be generous with context when reporting bugs or reviewing changes
- respect different levels of experience, language backgrounds, and workflows

## Unacceptable Behavior

- harassment, intimidation, discrimination, or personal attacks
- hateful or sexualized language or imagery
- doxxing or sharing private information without permission
- trolling, deliberate disruption, or bad-faith escalation
- repeated dismissive or hostile review behavior

## Scope

This Code of Conduct applies to:

- the repository
- issues
- pull requests
- discussions
- release and support spaces directly managed for TAP

## Reporting

If you experience or witness behavior that violates this Code of Conduct, contact the project maintainers through a private channel rather than a public issue whenever possible.

If a dedicated maintainer contact address has not been published yet, use the private contact method available through the repository host and include:

- what happened
- when and where it happened
- links, screenshots, or logs if available
- anything that would help maintainers respond safely

## Enforcement

Maintainers may take any action they believe is appropriate, including:

- asking for a change in behavior
- editing, hiding, or removing content
- temporarily restricting participation
- removing a contributor from project spaces

Maintainers will aim to review reports promptly, respect privacy as much as possible, and act in proportion to the situation.

## Project Standard

When in doubt, choose clarity, kindness, and professionalism.
